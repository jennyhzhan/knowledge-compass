# Compass Assistant 设计原则

本文档定义Compass系统的核心设计原则，供AI助手和用户参考。

## 核心理念

Compass是聚焦重点、尽量精简、结构化管理知识体系的助手。

- **Course**：每日精华总结，用户通过添加Task、Focus和Note让助手进一步了解自己的聚焦领域
- **Map**：核心知识连接，根据信息筛选和讨论重点总结的知识图谱
- **Cards**：详细内容载体，可以完整详尽

## Course文档生成原则

### 长度控制

- **Today's Summary**：约500字

  - 聚焦核心洞察和关键决策
  - 不重复卡片的详细内容
  - 提炼核心逻辑，不罗列所有事实
- **What's Next**：1-2件最重要的事

  - 最关键的时间节点（如：[重要事件时间]）
  - 最重要的行动项（如：[具体行动]）
  - 不需要完整的任务清单

### 内容筛选

**根据Focus筛选**：

- Focus明确关注的话题优先纳入
- 与Focus关键词匹配的卡片优先
- 不相关的fleeting卡片可以不纳入总结

**优先级排序**：

1. Insight卡片 > Fleeting卡片
2. 核心决策相关 > 一般信息
3. 核心逻辑 > 详细事实

## 知识图谱（map.canvas）生成原则

### 节点控制

- **总节点数**：5-10个
- **不需要包含所有卡片**
- 选择与Focus最相关、最有连接价值的

### 结构设计

- **中心节点**：当日核心主题
- **关键卡片**：insights + 1-2个最重要的fleeting
- **汇总节点**：提炼核心洞察的文本节点

## Note板块使用

### 位置和作用

- **位置**：Focus和Reference之间
- **性质**：用户的个人认知空间
- **作用**：影响AI后续分析方向

### 使用建议

**用户应该记录**：

- 对讨论话题的进一步思考和判断
- 个人的决策思路、偏好倾向
- 需要AI重点关注的独特角度

**AI应该如何使用**：

- 每天执行 `@navigation`时读取Note
- 分析时参考用户的独特视角
- 理解用户的思维方式和偏好
- 在sounding和course中体现用户关注点

## Sounding文档生成原则

### 内容选择

- **聚焦Focus话题**：不需要全面扫描所有新闻
- **深度优先**：宁可深入分析少数话题，不要浅显罗列大量信息
- **根据模版结构**：根据template文件夹中关于sounding的文档结构进行信息组织

## 卡片创建原则

### 何时创建

- **Insight**：深度分析、决策思考、核心洞察
- **Fleeting**：临时想法、新闻摘要、快速记录

### 命名规则

- 格式：`主题_YYYY-MM-DD.md`
- 不使用时间戳
- 同一天同主题合并到一个文件

### 内容详尽度

- **卡片可以详尽**：这是详细内容的载体
- **Course要精简**：这是每日总结和导航

## 工作流程

### 每日早晨（@navigation）

1. 读取昨日course（Task、Focus、Note）
2. 搜索Focus相关最新信息
3. 生成今日sounding（聚焦Focus）
4. 询问用户想探讨的话题

### 工作过程

- **@insight**：深度探讨，创建详细卡片
- **@fleeting**：快速记录，创建简要卡片
- 不是每张卡片都要纳入course

### 每日结束（@course）

1. 回顾今日讨论和卡片
2. **筛选核心内容**（不是全部）
3. 生成精简course（500字左右）
4. What's Next只列1-2个最关键事项

### 定期整理（@map）

- 生成知识图谱（5-10个节点）
- 突出核心主题和关键连接
- 不需要包含所有卡片

## 质量标准

### Course合格标准

- [ ] Today's Summary在500字左右
- [ ] 聚焦Focus相关的核心话题
- [ ] What's Next只有1-2个最关键事项
- [ ] 5分钟可以读完并明确行动

### Map合格标准

- [ ] 节点数5-10个
- [ ] 突出核心主题
- [ ] 连接关系清晰有意义
- [ ] 不包含所有卡片，只有精选

### Sounding合格标准

- [ ] 聚焦Focus话题，不全面扫描
- [ ] Analysis Update有明确建议
- [ ] 可操作性强

## 总结

**记住三个核心**：

1. **Course是精华，不是全集**

   - 500字总结核心
   - 1-2个关键行动
2. **Map是核心连接，不是全景图**

   - 5-10个节点
   - 精选最重要的卡片
3. **根据Focus筛选，不是面面俱到**

   - Focus相关的优先
   - Insight > Fleeting
   - 核心逻辑 > 详细事实

**详细内容在哪里？**

- 在卡片本身（insights、fleeting）
- 卡片可以详尽，course要精简
