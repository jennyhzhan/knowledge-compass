# Compass Assistant 使用指南

## 在Claude Code中使用（推荐）

你现在就可以直接在这个对话中使用所有功能！

### 快速启动

用任何简单的方式打招呼即可启动助手：

```
hi compass
卡帕斯你好
Compass助手
```

或者直接开始使用命令：

```
@navigation 启动
@insight 探讨某个话题
```

注意：使用 `@` 符号而非 `/` 避免与Claude skills命令冲突

我会自动：

1. 读取你昨天的course文档
2. 了解你当前的Focus
3. 搜索相关话题的最新信息
4. 生成今日的sounding文档

### 日常对话示例

**早晨启动：**

```
你：你好卡帕斯，@navigation
我：[读取昨日course] → [生成今日sounding] → 询问你想探讨哪个话题
```

**深度探讨：**

```
你：我想深入了解某个话题的最近进展
我：[搜索最新信息] → [分析和讨论] → 询问是否记录为知识卡片
```

**记录知识：**

```
你：是的，请记录为insight卡片
我：[生成卡片] → 保存到 logbook/YYYY-MM-DD/insights/
```

**结束一天：**

```
你：生成今日course
我：[回顾今日讨论] → [生成总结] → 保存course文档
```

## 所有命令

你可以使用 `@` 命令或自然语言：

### @navigation

**功能**: 开始新的一天
**流程**:

1. 读取昨日course
2. 提取Task和Focus
3. 搜索相关最新信息
4. 生成今日sounding草稿
5. 询问你想探讨的话题

**示例**:

```
你：@navigation
或：开始今天的工作
```

### @insight

**功能**: 深度探讨重点话题
**流程**:

1. 深入分析某个话题
2. 提供多角度观点
3. 讨论对你的影响
4. 在合适时询问是否记录

**示例**:

```
你：@insight 某个话题的最新进展
或：深入分析某个话题
```

### @fleeting

**功能**: 快速记录临时想法
**流程**:

1. 简短讨论某个话题
2. 记录碎片化信息
3. 保存为fleeting卡片

**示例**:

```
你：@fleeting 我看到一个有趣的应用场景
或：记录一个想法
```

### @card

**功能**: 手动创建知识卡片
**流程**:

1. 提供标题和内容
2. 选择类型（insight/fleeting）
3. 自动添加标签和元数据

**示例**:

```
你：@card 创建一个关于xxx的卡片
或：保存为知识卡片
```

### @map

**功能**: 生成今日知识图谱
**流程**:

1. 读取今日所有卡片
2. 分析卡片间的关联
3. 生成map.canvas文件

**示例**:

```
你：@map
或：生成知识图谱
```

### @harbor

**功能**: 归档到长期知识库
**流程**:

1. 将重要卡片整理归档
2. 按类别存入harbor对应文件夹
3. 更新索引

**示例**:

```
你：@harbor 将某个分析归档到companies
或：归档到harbor
```

### @analysis

**功能**: 生成深度分析报告
**流程**:

1. 读取相关历史卡片
2. 参考harbor中的frameworks
3. 生成结构化分析报告
4. 保存到harbor对应文件夹

**示例**:

```
你：@analysis 生成某个主题的深度分析
或：生成深度分析报告
```

### @course

**功能**: 生成今日course文档
**流程**:

1. 回顾今日讨论的话题
2. 保存course文档

**示例**:

```
你：@course
或：生成今日总结
```

## 自然对话

你不需要严格使用命令格式，我能理解自然语言：

```
你：我想了解一下某个话题的最新走势
我：[理解为 @insight] → 搜索并分析

你：把刚才的讨论记录下来
我：[理解为 @card] → 创建知识卡片

你：开始新的一天
我：[理解为 @navigation] → 执行navigation流程
```

## 典型一天的工作流

### 上午9:00

```
你：开始新的一天
我：读取昨日course → 生成今日sounding → "今天想聊哪个话题？"
```

### 上午9:30-11:00

```
你：聊聊某个话题
我：[深度讨论] → "要记录为insight吗？"
你：是的
我：[创建卡片] → "还想探讨什么？"
```

### 下午工作

```
你：看到一个有趣的应用
我：[记录fleeting] → "想深入研究吗？"
```

### 晚上总结

```
你：生成今日course
我：[回顾全天] → 生成course文档
```

## 最佳实践

1. **每天从 @navigation 开始** - 让我为你准备好一天的信息
2. **重点话题用 @insight** - 深度思考值得记录为insights
3. **临时想法用 @fleeting** - 先记下来，之后再整理
4. **定期执行 @map** - 看到知识之间的连接
5. **每天用 @course 结束** - 养成总结的习惯
6. **积累后执行 @analysis** - 将零散知识整合为系统分析

**提示**：你可以混用 `@命令` 和自然语言，我都能理解！

## 需要帮助？

随时问我：

- "现在是什么状态？" - 查看今日进度
- "我的Focus是什么？" - 回顾当前关注点
- "今天讨论了什么？" - 回顾今日话题
- "怎么使用xx命令？" - 获取帮助

## 注意事项

1. **所有文件自动保存到Obsidian** - 你可以在Obsidian中随时查看和编辑
2. **支持自然语言** - 不需要严格的命令格式
3. **智能询问** - 我会在合适的时机询问是否记录
4. **保持上下文** - 我记住我们讨论的内容，方便后续引用
